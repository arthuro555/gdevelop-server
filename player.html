Class: player

player(username, password, moderatoropt, socket_id)

Represents a Player. Authenticates players and store their login object_data and game object_data when online.

Constructor

new player(username, password, moderatoropt, socket_id)

Parameters:
Name Type Attributes Description
username string The username.
password string The password (will automatically be hashed).
moderator boolean <optional>
 Is the user an admin?
socket_id string Used to identify the current user without needing to trust the user controlled username.
Properties:
Name Type Description
object_data Array.<gdobject> An Array containing Objects and User data.
_password string The user password (Hashed).
_token Array.<string> An Array containing the Authentication tokens.
uuid string The player Unique ID to distinguish it from other instance.
username string The player Username.
online boolean If the player is not online, this flag will prevent object_data to be modified.
moderator boolean Modify this to true to let this player access Admin features (server-side).
socket_id string Permits to identify if a socket is the owner of an account.
